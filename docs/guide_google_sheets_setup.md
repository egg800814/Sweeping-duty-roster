## 2. 建立 Google 試算表

### 步驟 2.1：建立新試算表

1. 前往 [Google Sheets](https://sheets.google.com)。
2. 點擊左上角 **＋ 空白試算表**。
3. 將試算表命名為你喜歡的名稱（例如：`MDI 掃地排班系統`）。

### 步驟 2.2：建立「Schedule」工作表

預設的第一張工作表就是排班用的。請按以下步驟設定：

1. 雙擊底部的工作表分頁名稱（預設顯示「工作表1」）。
2. 將名稱改為 **`Schedule`**（注意大小寫）。
3. 在 **第一列（標題列）** 輸入以下欄位：

| 欄位 | A | B | C | D | E | F | G | H | I |
|------|---|---|---|---|---|---|---|---|---|
| 標題 | 日期 | 區域代碼 | 區域名稱 | 負責人員 | 確認1 | 確認2 | 確認3 | 確認4 | 今日負責人 |

> **💡 說明**：
> - A 欄：排班日期（格式 `yyyy-MM-dd`，例如 `2026-03-16`）
> - B 欄：區域 ID（例如 `a01`）
> - C 欄：區域名稱（例如 `1樓_大廳`）
> - D 欄：該區域的打掃人員，多人用逗號分隔（例如 `王小明, 李小華`）
> - E～H 欄：對應 D 欄各人員的確認狀態（由系統自動填入「已確認」）
> - I1 欄：今日排班負責人姓名（不是放在 I 欄的每一列，只放在 **I1** 格）

### 步驟 2.3：建立「StaffData」工作表（選用）

此工作表用於儲存今日出勤人員名冊，系統會自動建立。但你也可以預先建立：

1. 點擊底部的 **＋** 號新增工作表。
2. 將名稱改為 **`StaffData`**（注意大小寫）。
3. 在第一列輸入：

| 欄位 | A | B |
|------|---|---|
| 標題 | 姓名 | 性別 |

> **💡 說明**：不需手動填寫資料，系統同步時會自動更新此工作表。

## 3. 設定 Google Apps Script

### 步驟 3.1：開啟 Apps Script 編輯器

1. 在試算表上方選單點擊 **擴充功能** → **Apps Script**。
2. 會開啟一個新的 Apps Script 編輯器分頁。

### 步驟 3.2：貼上程式碼

1. 刪除編輯器中預設的所有程式碼（通常是一個空的 `myFunction`）。
2. 開啟本專案中的 `program/code.gs` 檔案。
3. **複製全部內容**，貼入 Apps Script 編輯器。
4. 點擊上方的 **💾 儲存**（或按 `Ctrl+S`）。

> **⚠️ 注意**：請確保完整複製 `code.gs` 的所有內容，不要遺漏任何程式碼。

### 步驟 3.3：設定專案名稱（選用）

在 Apps Script 編輯器左上角，可以將專案名稱改為「掃地排班系統 API」或你偏好的名稱。

## 4. 部署為 Web App

### 步驟 4.1：首次部署

1. 在 Apps Script 編輯器右上角，點擊 **部署** → **新增部署**。
2. 在「選取類型」旁邊，點擊 ⚙️ 齒輪圖示，選擇 **網頁應用程式**。
3. 填寫以下設定：

| 設定項目 | 值 |
|----------|------|
| 說明 | `v1`（或任何版本描述） |
| 執行身分 | **我**（你自己的 Google 帳號） |
| 具有存取權的使用者 | **任何人** |

4. 點擊 **部署**。
5. 系統可能會要求授權，點擊 **授權存取**。

### 步驟 4.2：處理授權警告

首次授權時 Google 可能會顯示「這個應用程式未經 Google 驗證」的警告：

1. 點擊左下角的 **進階**。
2. 點擊 **前往「掃地排班系統 API」（不安全）**。
3. 點擊 **允許**。

### 步驟 4.3：複製部署 URL

1. 部署成功後，畫面會顯示一組 **網頁應用程式 URL**。
2. 格式為：`https://script.google.com/macros/s/AKfycb.../exec`
3. **複製此 URL**，這是你的 GAS API 網址。

> **⚠️ 非常重要**：請妥善保存此 URL，後續設定排班系統時需要使用。

## 5. 將 API URL 設定到排班系統

你可以選擇以下 **任一方式** 來設定：

### 方法 A：透過管理介面設定（推薦）

1. 打開排班系統 `frontend/setup.html`。
2. 點擊頂部的 **⚙️ 資料管理** 分頁。
3. 在「🔗 Google Sheets API 設定」區塊中，將剛才複製的 URL 貼入輸入框。
4. 點擊 **💾 儲存**。
5. 確認顯示「✅ 已儲存並即時生效」。

### 方法 B：修改 `data/data.json` 設定檔

1. 用文字編輯器開啟 `data/data.json`。
2. 找到 `"gasApiUrl"` 欄位。
3. 將值替換為你的 GAS API URL：

```json
{
  "gasApiUrl": "https://script.google.com/macros/s/你的部署ID/exec",
  ...
}
```

4. 儲存檔案。

> **💡 提示**：方法 A 更方便快速；方法 B 適合初次設定或多裝置統一管理。

## 6. 驗證連線

### 6.1 測試 API 是否正常

1. 打開瀏覽器，直接貼上你的 GAS API URL。
2. 應該會看到類似以下的 JSON 回應（試算表若為空則回傳空陣列）：

```json
[]
```

若看到一串 JSON 資料或空陣列，代表 API 部署成功。

### 6.2 測試排班系統連線

1. 打開排班系統 `frontend/setup.html`。
2. 進行以下操作：
   - 勾選一些出勤人員。
   - 點擊「🎲 自動排班」。
   - 觀察是否出現「雲端同步成功」的綠色提示。
3. 回到 Google Sheets 檢查 Schedule 工作表，應該能看到新寫入的排班資料。

### 6.3 測試打掃確認頁面

1. 打開 `frontend/viewer.html`。
2. 應該能看到剛才排好的區域卡片。
3. 點擊任一人名按鈕，觀察是否成功顯示「✓ 已確認」。
4. 回到 Google Sheets 確認 E～H 欄是否出現「已確認」。

## 7. 後續維護

### 7.1 更新 Apps Script 程式碼

如果需要更新 `code.gs`：

1. 在 Apps Script 編輯器中貼上新的程式碼。
2. 儲存。
3. 點擊 **部署** → **管理部署**。
4. 點擊右上角的 ✏️ **鉛筆圖示**。
5. 在「版本」下拉選單選擇 **建立新版本**。
6. 點擊 **部署**。

> **⚠️ 重要**：更新程式碼後，**必須建立新版本並重新部署**，否則 API 仍然使用舊版程式碼。如果更新後 URL 沒變，則排班系統不需要重新設定。

### 7.2 更換 Google 試算表

如需更換為新的試算表（例如新學年或新季度）：

1. 按照 [步驟 2](#2-建立-google-試算表) 建立新試算表。
2. 按照 [步驟 3](#3-設定-google-apps-script) 設定 Apps Script。
3. 按照 [步驟 4](#4-部署為-web-app) 部署並取得新 URL。
4. 按照 [步驟 5](#5-將-api-url-設定到排班系統) 更新排班系統的 API URL。

### 7.3 試算表權限管理

- Google Sheets 的「共用」功能可以讓其他管理者直接查看雲端資料。
- 但 **不要** 手動修改 Schedule 工作表的格式或合併儲存格，可能導致 API 讀寫異常。
- StaffData 工作表的內容由系統自動管理，不需手動編輯。

## 8. 疑難排解

### ❌ API URL 貼上後顯示「⚠️ 格式錯誤」

**原因**：URL 不是以 `https://script.google.com/` 開頭。

**解決方法**：
- 確認你複製的是 **部署後的網頁應用程式 URL**，而非 Apps Script 編輯器的 URL。
- 正確格式：`https://script.google.com/macros/s/.../exec`

---

### ❌ 同步時顯示「雲端同步發生錯誤」

**可能原因與解決方法**：

| 可能原因 | 解決方法 |
|----------|----------|
| API URL 已過期 | 重新部署 Apps Script 並更新 URL |
| 試算表被刪除 | 建立新的試算表並重新設定 |
| Google 帳號權限變更 | 重新在 Apps Script 中授權 |
| 網路問題 | 檢查網路連線後重試 |

---

### ❌ viewer.html 顯示「未設定有效的 API URL」

**解決方法**：
1. 確認 `data/data.json` 中的 `gasApiUrl` 已填入正確的 URL。
2. 或在 `setup.html` 的「資料管理」分頁中設定 URL。

---

### ❌ 部署後出現「找不到今日符合的區域代號」

**原因**：試算表中還沒有今日的排班資料。

**解決方法**：先在 `setup.html` 執行一次「🎲 自動排班」，讓資料同步到試算表。

---

### ❌ 更新 Apps Script 後行為沒變

**原因**：修改程式碼後沒有建立新版本部署。

**解決方法**：
1. 前往 **部署** → **管理部署**。
2. 點擊 ✏️ 編輯。
3. 版本選擇「建立新版本」。
4. 點擊「部署」。

---

### ❌ 授權時出現「這個應用程式遭到 Google 封鎖」

**可能原因**：使用公司管理的 Google Workspace 帳號，管理員限制了第三方應用。

**解決方法**：
- 聯繫 Google Workspace 管理員放行此應用程式。
- 或改用個人 Google 帳號部署。
