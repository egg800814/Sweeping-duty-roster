## 1. 進入確認系統

1. 開啟 `index.html`（排班系統入口）。
2. 點擊左側的「🧹 一般打掃人員」卡片。
3. 系統會自動從 Google Sheets 載入今日排班資料。

---

## 2. 查看今日排班

進入後，畫面會以 **卡片** 形式列出今日所有打掃區域：

每張卡片包含：
- **區域代號**（如「區域 a01」）
- **區域名稱**（如「1樓_大廳」）
- **負責人員按鈕**（顯示姓名與性別標示）

### 特殊狀態

| 狀態 | 說明 |
|------|------|
| **💤 今日免掃** | 該區域今日無需清潔（顯示「⛱️ 此區域今日無需清潔」） |
| **未確認** | 按鈕顯示人員姓名，可點擊確認 |
| **✓ 已確認** | 按鈕變為綠色並顯示「✓ 姓名 - 已確認」 |

### 性別標示

- ♂ 男性人員按鈕為 **藍色系**
- ♀ 女性人員按鈕為 **粉紅色系**

---

## 3. 確認打掃完成

1. 找到你被分配的區域卡片。
2. 點擊 **你的名字按鈕**。
3. 按鈕會短暫顯示「傳送中...」。
4. 成功後變為 **綠色** 並顯示「✓ 姓名 - 已確認」。
5. 確認後按鈕會被鎖定，無法重複點擊。

> **⚠️ 注意**：請確實完成打掃後再點擊確認按鈕。

---

## 4. 畫面說明

### 自動更新

- 頁面每 **15 秒** 會自動重新抓取最新資料。
- 若排班負責人在管理端調整了分配，你的畫面會自動更新。
- 不需要手動重新整理頁面。

### 載入狀態

| 畫面訊息 | 意義 |
|----------|------|
| 「資料讀取中...」 | 正在從雲端取得資料 |
| 「未設定有效的 API URL」 | 系統尚未設定 Google Sheets 連線 |
| 「連線失敗，請檢查網路或 API 設置」 | 網路異常或 API 設定有誤 |
| 「今日打掃區域尚未安排」 | 今天還沒有排班資料 |

## 5. 常見問題

### Q：頁面顯示「未設定有效的 API URL」？
請聯繫排班負責人，確認 Google Sheets API 是否已正確設定。

### Q：按鈕按下去顯示「傳送失敗」？
1. 檢查網路連線是否正常。
2. 稍後再試一次。
3. 若持續失敗，請聯繫排班負責人。

### Q：我找不到我的名字？
1. 可能今天被分配到不同區域，請仔細檢查所有卡片。
2. 若確實不在任何區域中，聯繫排班負責人確認是否漏排。

### Q：已確認後發現打掃區域搞錯了怎麼辦？
確認狀態無法由打掃人員自行撤銷，請聯繫排班負責人處理。

### Q：手機上可以使用嗎？
可以。系統支援手機瀏覽器，介面會自動適應螢幕大小。
