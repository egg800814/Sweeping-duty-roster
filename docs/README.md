# 🧹 MDI 掃地排班系統 V13.0

公司每日清掃區域智慧排班工具，支援自動分配、雲端同步與即時打掃確認。

---

## 🚀 快速開始

本系統為純前端網頁應用，無需安裝任何軟體。

1. **線上使用**：部署至 GitHub Pages 或內部伺服器後，透過網址開啟。
2. **單機使用**：直接用瀏覽器開啟 `index.html`。
3. **首次使用**：系統自動從 `data/data.json` 載入預設人員與區域設定。

---

## 🏠 系統入口

開啟 `index.html` 後會看到兩個入口：

| 入口 | 對象 | 功能 |
|------|------|------|
| 🧹 **一般打掃人員** | 所有員工 | 查看今日分配區域、完成線上打掃確認 |
| 🛠️ **今週の朝掃除責任者** (ADMIN) | 排班負責人 | 管理排班、人員、區域、輪值及系統設定 |

## 📋 核心功能

### 每日排班（管理員）
- 勾選出勤人員 → 一鍵自動排班
- 演算法自動考慮：區域優先級、性別限制、樓層限制、歷史公平性
- 支援拖曳調整人員分配、追加遲到人員
- 排班結果自動同步至 Google Sheets

### 打掃確認（一般人員）
- 查看今日負責區域
- 點擊按鈕完成確認，狀態即時同步至雲端
- 每 15 秒自動更新畫面

### 人員管理
- 新增 / 編輯 / 刪除人員
- 設定所屬部門、性別、角色權重（如部長）、樓層限制、排除區域、是否參與輪值

### 部門與角色管理 (V13.0 新增)
- 動態新增、編輯、刪除部門清單，並可設定排序優先級
- 動態建立職稱角色，可設定權重數值，權重越小排序越前（如 `1` 為最高主管）

### 區域管理
- 設定清掃區域的優先級（每日必掃 / 彈性 / 可選）
- 設定性別限定、人數範圍、假日前加人

### 輪值設定
- 每週負責人自動輪替
- 支援手動調整順序、基準日設定
- 負責人不參與掃地分配

### 資料管理
- 匯出 / 匯入 JSON 備份
- Google Sheets API URL 設定
- 資料自動從 `data/data.json` 同步

## 🔗 Google Sheets 整合

系統透過 Google Apps Script (GAS) 與 Google Sheets 雙向同步：

- **管理端 → 雲端**：排班結果、出勤名單自動寫入試算表
- **雲端 → 確認端**：打掃人員從試算表讀取分配並回報確認狀態

> 詳細設定步驟請參閱 [Google 試算表重新設定指南](guide_google_sheets_setup.md)

## ❓ 常見問題

**Q：換瀏覽器後資料不見了？或剛編輯的資料重新整理後不見了？**  
系統每次啟動（重新整理網頁）皆會**強制**清空本機暫存，並完全從 GitHub 上的 `data/data.json` 重新下載覆蓋最新設定，以確保所有人看到的資料完全同步對齊。
若您是管理員，在完成人員、部門等異動後，**必須到資料管理點擊「備份資料(匯出JSON)」**，並手動覆蓋回程式碼的 `data/data.json` 後推上 Git 上傳，新資料才會真正存檔生效並派發給所有人。

**Q：某人臨時請假怎麼處理？**  
在排班頁面取消勾選該人員，重新點擊「🎲 自動排班」即可。

**Q：排班後忘記儲存？**  
排班結果已自動同步至 Google Sheets，但本地歷史紀錄需點「💾 儲存排班」才會保存。

**Q：如何更新 Google Sheets API 網址？**  
進入 `setup.html` → ⚙️ 資料管理 → 貼上新 URL → 💾 儲存。
